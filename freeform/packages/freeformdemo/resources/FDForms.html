Forms Demo

Freeform Framework greatly simplifies form handling. It uses special Form class to abstract the input form, its fields and processing action. The form fields are special classes that can validate themselves. Besides, the HTML package will automatically display warning messages for you if the form has been filled with mistakes, and will take care of setting the field values - so that the page designers will not have to worry about the action attribute of the form and about inserting the field values manually. The encapsulation of an input form into a separate class gives you the power of having multiple forms on the same page as well as having same form on multiple pages. Page designer can even create a separate template for the form and include it on every page that will have that form!

You can test the form by trying to enter bad values: first try to enter all values incorrectly, then enter one field correctly and so on. You will see the form save correct input (try switching the radio buttons and checking the box between incorrect submissions). Also note that the list of errors will also change. Please use the 'View Source' and 'View Template' links above to see how easy to make the template and to build the form.

In general, you will need to construct a Form object, add fields to it and display it. You dipslay a form very much like you did with simple HTML forms, but replacing the form and input tags with their HTMLShowForm and HTMLInput counterparts. The HTMLIfInputFieldErrors and HTMLIfInputFieldError tags are conditional tags that will display their contents only if form fields were incorrectly filled.
